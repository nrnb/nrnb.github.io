Cytoscape 3.2 Launch Challenge November 2014

Winners

Best example of using Cytoscape for Biology

  • 1st - Thomas Kelder: The Food - Genomics - Disease landscape: A network linking food ingredients with human disease through underlying molecular interactions

  • 2nd - Mark Grimes: Co-cluster correlation network of phoshporylated proteins in neuroblastoma samples.

  • 3rd - Kalaiarasan Ponnusamy: Crosstalk among the modules and hubs in the apoptosis protein-protein interaction network.

Most valuable feature to add to Cytoscape

  • 1st - Gordon Bean

    An essential, innovative feature of Cytoscape has been its ability to provide the use with extensive control over the qualities of network visualizations; while on the other hand, another important feature has been to provide users with attractive visualizations as built-in options. Because sophisticated visualizations depend on mapping visual style elements to data, providing advanced built-in visual styles has been a challenge. However, providing a “wizard” user interface to simple styles based on common data-driven visualizations would: teach the user about basic visual style techniques, such as mapping discrete colors to discrete node classes, or using edge weights to denote edge score or strength,

    • Ask the user to select attributes that match these criteria (Cytoscape can scan data for each attribute to loosely determine what attributes might be appropriate - i.e. continuous, discrete, pass-through, etc),
    • Automatically bind these attributes to sensible style properties that match the rest of the style theme.

  • 2nd - Mark Fortner:

    Suggest using Oracle JavaFX packager for Cytoscape installers, which package the required version of Java with the application, thus cutting down on the amount of headscratching by users.

  • 3rd - Srikanth Bezawada:

    • Case studies on social network data, visualizing transportation networks data.
    • Implement generic feature to allow an app to be run on a large set of networks and allow for efficient extraction of app results to compare networks on a large scale.

  • 3rd - Thomas Pfau:

    Implement edges with multiple sources and targets to allow for programmatic representation of more complex interactions. For example, they could be used to represent Gene-Protein-Reaction relationships without necessitating the introduction of Enzyme subunits.
